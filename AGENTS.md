You are building a Docusaurus 3 client engagement knowledge base template for a coaching or consulting practice.

## Context

- The project is at the repository root.
- A vanilla Docusaurus 3 scaffold (TypeScript, classic preset) is installed and builds clean.
- Node.js 20 LTS and Yarn 1.x are the runtime and package manager.
- Do not run any terminal commands. No yarn, npm, npx, or shell commands. Only create and edit files. The developer handles all installs and builds locally.

## Source of Truth: Design System

The design system lives in `src/css/custom.css`. Read this file for the canonical reference on colors, typography, spacing, and component patterns. When in doubt about a design decision, read the source — do not invent.

Read the CLAUDE.md files at each directory level for context specific to that part of the project. They form a layered context system — root establishes constraints, descendant files add specificity.

## What You Are Building

A template repository cloned for every new client engagement. Each clone receives a `client.json` file. The template must be fully configurable from `client.json` — no hardcoded client data.

Each client site deploys to its configured `siteUrl` via GitHub Actions. Set `FTP_HOST`, `FTP_USER`, and `FTP_PASS` as GitHub Actions secrets, or modify `.github/workflows/deploy.yml` for your deployment target.

The end product is an editorial knowledge base — clean typography, warm design, professional layout — configured for a specific client engagement.

## Engagement Types

The template supports these engagement types without modification:

| Type | Sidebar Pattern | Content Structure |
|------|----------------|-------------------|
| Cohort coaching | Phase categories + Cohort section | Phase folders + cohort/ |
| Individual coaching | Phase categories | Phase folders |
| Grounded Growth | Custom phases | Custom phase folders |
| Org consulting | Custom phases from proposal | Custom phase folders |
| Workshop / retreat | Pre-work / Facilitation / Follow-up | 3-phase folders |
| Retainer / advisory | Monthly categories (YYYY-MM) | Chronological folders |

## Design System

### Typography
- **Headlines:** 'DM Serif Display', serif — all headings, section titles, participant names
- **Body / UI:** 'Lato', sans-serif — body text, sidebar, nav, paragraphs
- Import via Google Fonts in `src/css/custom.css`. Use the weights already defined there.

### Color System
- **Warm structural palette (fixed):** backgrounds, text colors, borders — the template's base design layer
- **Client accent colors (configurable):** `primaryColor` and `secondaryColor` from `client.json` override accent colors only
- **Dark mode:** Deep dark backgrounds, warm cream text, editorial character preserved

### Readability
- Base font size: at least 17px
- Body line-height: at least 1.75
- Sidebar link text: at least 0.9rem
- Content area max-width: ~760px

## Key Constraints

- `routeBasePath: '/'` in docs config. No `src/pages/index.tsx`. No blog.
- Every autogenerated sidebar category needs `link: { type: 'generated-index' }`.
- `editUrl: undefined` — client sites should not expose repo links.
- No secondary TOC panel — sidebar is the only navigation.
- Footer: configurable via `client.json` → `tagline`.

## Content Guidelines

- Human-centered language
- Strengths-based framing
- Name systems, not symptoms
- Warm, direct, mission-oriented tone
- The knowledge base is a gift the client walks away owning
